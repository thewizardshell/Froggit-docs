title: Filosofía
description: Los principios de diseño detrás de Froggit y por qué existe.
---

Froggit nació de una frustración simple: Git es poderoso, pero su interfaz de línea de comandos puede ser intimidante, especialmente para tareas cotidianas que deberían ser rápidas y directas.

Muchas herramientas intentan resolver esto agregando capas de complejidad. Froggit toma el camino opuesto.

## Principios guía

**Mantenerlo simple.** Cada decisión de diseño en Froggit prioriza la claridad sobre la cantidad de funciones. La interfaz muestra solo lo que necesitas en cada momento, sin menús profundos ni opciones ocultas que requieran un manual.

**Enfocarse en el ciclo.** El flujo de trabajo más común en Git es: ver cambios, preparar archivos, escribir un commit, y enviar. Froggit optimiza este ciclo para que sea lo más fluido posible.

**Ser moderno.** Froggit usa herramientas contemporáneas de renderizado de terminal para ofrecer una experiencia visual limpia con colores, iconos y animaciones sutiles que ayudan sin distraer.

**Minimizar la fricción.** Cada interacción está diseñada para requerir el menor esfuerzo posible. Los controles aparecen donde los necesitas, los atajos son intuitivos, y los mensajes de estado se limpian solos.

**Ser accesible.** Froggit está diseñado tanto para quien usa Git por primera vez como para quien lleva años trabajando con él. No asume conocimiento previo, pero no limita a quienes lo tienen.
